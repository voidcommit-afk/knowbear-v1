# Agent/Skills.md: Exhaustive Guide for Agentic IDE Skills## IntroductionThis document defines the skills for the Agentic IDE, an autonomous AI-powered system for software development. Agents perform tasks collaboratively, ensuring all generated code meets the highest standards:- **High Quality**: Readable, maintainable, efficient, and scalable.- **Zero Vulnerabilities**: No known security flaws (e.g., via static analysis).- **Max Security**: Encryption, access controls, and threat modeling integrated.- **No Future Bugs**: Defensive programming, extensive testing, and forward-compatible design.- **Prod-Grade**: Ready for deployment with logging, monitoring, error handling, and compliance (e.g., GDPR, HIPAA where applicable).All skills must adhere to:- **Languages/Frameworks**: Support major ones (e.g., Python, JavaScript, Java, Go) with modular extensions.- **Tools Integration**: Use linters (ESLint, Pylint), scanners (SonarQube, Snyk), testers (JUnit, Jest), and CI/CD (GitHub Actions).- **Metrics**: Aim for 100% test coverage, zero critical lint errors, Cyclomatic Complexity &lt; 10 per function.- **Documentation**: Inline comments, READMEs, and API docs (e.g., JSDoc, Sphinx).- **Versioning**: Semantic Versioning (SemVer) for changes.Agents must log all actions, rollback on failures, and escalate to human review if confidence &lt; 95%.## Core Skills### 1. Project Planning and Architecture Design**Description**: Agent analyzes requirements to create a high-level architecture, including modules, data flows, and tech stack.**Process**:1. Parse user requirements (e.g., via natural language).2. Generate UML diagrams (class, sequence, ER).3. Select stack (e.g., React for FE, Node.js for BE, PostgreSQL for DB).4. Define microservices/monolith based on scalability needs.**Quality Requirements**:- Modular design following SOLID principles.- Scalable to 10x load without redesign.- Readable blueprints with clear separation of concerns.**Security Mandates**:- Incorporate threat modeling (STRIDE).- Mandate HTTPS, JWT/OAuth for auth.- No hard-coded secrets; use env vars/secrets managers (e.g., Vault).**Anti-Bug Strategies**:- Future-proof with extensible interfaces/abstract classes.- Validate assumptions with mock data.- Ensure compatibility with future versions (e.g., deprecate old APIs gracefully).**Prod-Grade Checks**:- Include deployment diagrams (Kubernetes/Docker).- Estimate costs (AWS/GCP calculators).- 100% documentation of architecture decisions.### 2. Code Generation**Description**: Generate complete, functional code for FE/BE components based on specs.**Process**:1. Break down into functions/classes.2. Write code in idiomatic style.3. Integrate with existing codebase via git diffs.**Quality Requirements**:- Clean Code: Short functions (&lt;50 lines), meaningful names, no duplication.- Performance: O(n) optimizations where possible.- Accessibility (FE): ARIA attributes, semantic HTML.**Security Mandates**:- Sanitize inputs (e.g., DOMPurify for XSS).- Use prepared statements for SQL (no injections).- Encrypt sensitive data (AES-256).- Rate limiting and CSRF tokens.**Anti-Bug Strategies**:- Type safety (TypeScript/Flow for JS, type hints in Python).- Error boundaries (try-catch, custom exceptions).- Version pinning for dependencies to avoid breaking changes.**Prod-Grade Checks**:- Modular exports for reusability.- Logging (e.g., Winston/Sentry) at INFO/ERROR levels.- Graceful degradation (e.g., offline support in FE).### 3. Debugging**Description**: Identify and fix bugs in existing or generated code.**Process**:1. Reproduce issue with logs/traces.2. Use binary search or rubber-duck debugging.3. Apply fixes and verify.**Quality Requirements**:- Minimal changes to preserve original intent.- Refactor if complexity increases.**Security Mandates**:- Scan for introduced vulns (e.g., buffer overflows).- Ensure fixes don't leak data.**Anti-Bug Strategies**:- Add assertions/invariants.- Prevent regressions with pinned tests.- Handle edge cases (nulls, extremes).**Prod-Grade Checks**:- Update monitoring (Prometheus/Grafana).- Log fix details for audit trails.### 4. Refactoring**Description**: Improve code structure without changing behavior.**Process**:1. Analyze metrics (duplication, complexity).2. Apply patterns (e.g., Factory, Observer).3. Test before/after.**Quality Requirements**:- Improve readability (e.g., extract methods).- Reduce tech debt.**Security Mandates**:- Strengthen weak points (e.g., add input validation).- Audit for deprecated crypto.**Anti-Bug Strategies**:- Preserve invariants via tests.- Modularize for future extensions.**Prod-Grade Checks**:- Performance benchmarks pre/post.- Update docs inline.### 5. Testing**Description**: Create and run unit, integration, E2E, and security tests.**Process**:1. Generate tests covering happy/sad paths.2. Run in CI pipeline.3. Report coverage.**Quality Requirements**:- 100% branch coverage.- Mock dependencies for isolation.**Security Mandates**:- Fuzz testing for inputs.- Penetration tests (e.g., OWASP ZAP).**Anti-Bug Strategies**:- Mutation testing to ensure tests catch changes.- Test for concurrency (race conditions).**Prod-Grade Checks**:- Integrate with CI/CD for auto-deployment on pass.- Load testing (JMeter).### 6. Security Auditing**Description**: Scan and remediate vulnerabilities.**Process**:1. Run static/dynamic analysis.2. Prioritize CVEs.3. Apply patches.**Quality Requirements**:- Zero high/critical findings.- Compliance with standards (PCI-DSS).**Security Mandates**:- Full SBOM (Software Bill of Materials).- Least privilege access.- Multi-factor auth enforcement.**Anti-Bug Strategies**:- Dependency updates scheduled.- Canary tokens for breach detection.**Prod-Grade Checks**:- Audit logs immutable.- Incident response plan integrated.### 7. Performance Optimization**Description**: Tune code for speed, memory, and scalability.**Process**:1. Profile (e.g., Chrome DevTools, pprof).2. Optimize bottlenecks.3. Benchmark.**Quality Requirements**:- Meet SLAs (e.g., &lt;100ms response).- Efficient algorithms.**Security Mandates**:- No DoS vulnerabilities (e.g., regex backtracking).- Secure caching (Redis with auth).**Anti-Bug Strategies**:- Monitor for degradation over time.- Scale tests (horizontal/vertical).**Prod-Grade Checks**:- Auto-scaling configs.- CDN integration for FE.### 8. Documentation Generation**Description**: Create comprehensive docs.**Process**:1. Extract from code/comments.2. Generate API specs (OpenAPI).3. Write user guides.**Quality Requirements**:- Clear, concise, with examples.- Versioned docs.**Security Mandates**:- Redact sensitive info.- Secure doc hosting.**Anti-Bug Strategies**:- Link docs to code for auto-updates.- Validate examples run error-free.**Prod-Grade Checks**:- SEO-optimized for internal wikis.- Accessibility (WCAG 2.1).### 9. Version Control and Collaboration**Description**: Manage git operations, PRs, and merges.**Process**:1. Branch, commit, merge.2. Review changes.3. Resolve conflicts.**Quality Requirements**:- Atomic commits.- Conventional Commits format.**Security Mandates**:- Signed commits (GPG).- Branch protection rules.**Anti-Bug Strategies**:- Pre-commit hooks for tests/lints.- Rebase for clean history.**Prod-Grade Checks**:- CI triggers on push.- Release notes auto-generated.### 10. Deployment and Monitoring**Description**: Deploy code and set up observability.**Process**:1. Build artifacts (Docker images).2. Deploy to env (dev/stg/prod).3. Monitor metrics.**Quality Requirements**:- Zero-downtime deploys (blue-green).- Rollback plans.**Security Mandates**:- Secrets scanning in pipelines.- HTTPS-only.**Anti-Bug Strategies**:- Canary releases.- Chaos engineering tests.**Prod-Grade Checks**:- Alerts on anomalies (Pagerduty).- Compliance certifications.### 11. Compliance and Auditing**Description**: Ensure legal/regulatory adherence.**Process**:1. Scan for PII.2. Generate reports.3. Remediate issues.**Quality Requirements**:- Traceable changes.**Security Mandates**:- Data anonymization.- Consent management.**Anti-Bug Strategies**:- Automated compliance tests.**Prod-Grade Checks**:- Audit trails for SOX/GDPR.### 12. Extension and Customization**Description**: Add new skills or adapt existing ones.**Process**:1. Define plugin interface.2. Integrate user-defined agents.**Quality Requirements**:- Backward-compatible APIs.**Security Mandates**:- Sandbox extensions.**Anti-Bug Strategies**:- Version locking.**Prod-Grade Checks**:- Marketplace for verified plugins.## Appendices- **Error Handling Guidelines**: Always use structured errors with codes/messages.- **Best Practices References**: OWASP Top 10, CWE, NIST SP 800-53.- **Update Policy**: Review this file quarterly for new threats/standards.
